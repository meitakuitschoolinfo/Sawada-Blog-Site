# CRUD図（機能 × テーブル マトリクス）

| システム名 | Blog アプリケーション |
| --- | --- |

各機能が、各テーブルに対して行う操作（C=作成 / R=参照 / U=更新 / D=削除）を示します。

## 1. CRUD マトリクス

| 機能ID | 機能名 | users | blogs |
| --- | --- | :---: | :---: |
| A1-02 | ユーザー登録 | C R | - |
| A1-04 | ログイン | R | - |
| A1-05 | ログアウト | - | - |
| B1-01 | ブログ一覧表示 | - | R |
| B1-03 | ブログ登録 | - | C R |
| B1-04 | ブログ編集画面表示 | - | R |
| B1-05 | ブログ更新 | - | R U |
| B1-06 | 画像編集画面表示 | - | R |
| B1-07 | 画像更新 | - | R U |
| B1-08 | 削除候補一覧表示 | - | R |
| B1-09 | 削除記事詳細表示 | - | R |
| B1-10 | ブログ削除 | - | D |

> 凡例: C=Create（登録） / R=Read（参照） / U=Update（更新） / D=Delete（削除） / -=操作なし

---

## 2. 操作内容の補足

| 機能ID | テーブル | 操作 | 内容 |
| --- | --- | --- | --- |
| A1-02 | users | R | `findByEmail` でメール重複を確認 |
| A1-02 | users | C | 重複が無ければ `save` で新規登録 |
| A1-04 | users | R | `findByEmailAndPassword` で認証 |
| B1-01 / B1-08 | blogs | R | `findByUserId` でユーザーの記事一覧を取得 |
| B1-03 | blogs | R | `findByBlogTitleAndRegisterDate` で重複確認 |
| B1-03 | blogs | C | 重複が無ければ `save` で新規登録 |
| B1-04 / B1-06 / B1-09 | blogs | R | `findByBlogId` で対象記事を取得 |
| B1-05 | blogs | R, U | `findByBlogId` で取得後、`save` でテキスト項目を更新 |
| B1-07 | blogs | R, U | `findByBlogId` で取得後、`save` で画像名を更新 |
| B1-10 | blogs | D | `deleteByBlogId` で削除（`@Transactional`） |

---

## 3. テーブル別 影響機能サマリ

| テーブル | 作成(C) | 参照(R) | 更新(U) | 削除(D) |
| --- | --- | --- | --- | --- |
| users | A1-02 | A1-02, A1-04 | （なし） | （なし） |
| blogs | B1-03 | B1-01, B1-03, B1-04, B1-05, B1-06, B1-08, B1-09 | B1-05, B1-07 | B1-10 |

> 備考: `users` テーブルには更新（U）・削除（D）機能が存在しません（ユーザー情報変更・退会機能は未実装）。今後の拡張候補です。
