# メソッド定義書

| システム名 | Blog アプリケーション |
| --- | --- |

本書は、本アプリの全 **DAO / Service / Controller** メソッドの仕様を、メソッドごとに記載したものです。
各メソッドは「メソッド名・クラス名・ファイル名・パッケージ名・定義・可視性・static・概要・引数・返却値・詳細記述・例外事項・備考」の項目で記述します。

> 凡例
> - 可視性: `public` / `private` など
> - static: 静的メソッドかどうか（`-` は非 static）
> - 引数／返却値が無い場合は「なし」と記載

---

# Ⅰ. DAO（データアクセス層）

DAO は `JpaRepository` を継承したインターフェースであり、実装は Spring Data JPA がメソッド名から自動生成します。

| パッケージ名 | blog.ex.service |
| 定義 | ユーザー名・メール・パスワードを受け取り、新規ユーザーアカウントを作成する。 |
| 可視性 | public |
| static | - |

**概要**：メールアドレスの重複が無ければ新規ユーザーを登録する。

**引数**

| 項番 | 引数名 | 型 | 説明 |
| --- | --- | --- | --- |
| 1 | userName | String | ユーザー名 |
| 2 | email | String | メールアドレス |

**返却値**

| 項番 | 型 | 説明 |
| --- | --- | --- |
| 1 | boolean | 登録成功:true / メール重複により失敗:false |

**詳細記述**
1. 現在日時を `LocalDateTime.now()` で取得し registerDate とする。
2. `userDao.findByEmail(email)` で既存ユーザーを検索する。
3. 該当が無い（null）場合、新規 UserEntity を作成し `userDao.save()` で保存して true を返す。
4. 既に存在する場合は false を返す。

| --- | --- | --- |
| 1 | データベースアクセスエラー | 本メソッドでは処理せず、呼び出し元へ伝播する。 |

**備考**：パスワードは平文で保存（学習用）。ハッシュ化のサンプルはソース内コメント参照。
