```mermaid
flowchart LR

    Controller -- "Model + ビュー名" --> View
    View -- "HTML レスポンス" --> Browser
    Browser -- "静的ファイル取得" --> Static

    Controller --> Service
    Service --> Dao
    Dao -- "JDBC (Hibernate)" --> DB
    Controller -- "画像アップロード保存" --> FileStore
```

## 2. 技術スタック構成

```mermaid
flowchart TB
    subgraph Presentation["プレゼンテーション"]
        TH["Thymeleaf"]
        MVC["Spring MVC"]
    end
    subgraph Business["ビジネスロジック"]
        SV["Spring @Service"]
    end
    subgraph DataAccess["データアクセス"]
        JPA["Spring Data JPA"]
        HB["Hibernate (JPA 実装)"]
    end
    subgraph Infra["インフラ"]
        PG["PostgreSQL"]
    end

    MVC --> SV --> JPA --> HB --> PG
    MVC --- TH
```

| 層 | 採用技術 | バージョン／補足 |
| --- | --- | --- |
| 言語 | Java | 17 |
| フレームワーク | Spring Boot | 3.0.4 |
| Web | Spring MVC | spring-boot-starter-web |
| ビュー | Thymeleaf | spring-boot-starter-thymeleaf |
| 永続化 | Spring Data JPA / Hibernate | spring-boot-starter-data-jpa |
| DB | PostgreSQL | localhost:5433 / blog_demo |
| 補助 | Lombok | アクセサ等の自動生成 |
| ビルド | Maven | Maven Wrapper 同梱 |
| テスト | JUnit5 / Mockito / MockMvc | spring-boot-starter-test |

## 3. リクエスト処理フロー（共通）

```mermaid
sequenceDiagram
    participant B as ブラウザ
    participant DS as DispatcherServlet
    participant C as Controller
    participant S as Service
    participant D as DAO(JPA)
    participant DB as PostgreSQL
    participant V as Thymeleaf

    B->>DS: HTTP リクエスト
    DS->>C: ハンドラーメソッド呼出
    C->>S: 業務処理依頼
    S->>D: データ操作
    D->>DB: SQL 実行
    DB-->>D: 結果
    D-->>S: エンティティ
    S-->>C: 処理結果
    C->>V: Model + ビュー名
    V-->>DS: HTML 生成
    DS-->>B: HTTP レスポンス
```

## 4. デプロイ／実行構成（ローカル）

| 要素 | 設定値 |
| --- | --- |
| アプリ起動ポート | 8080（Spring Boot 既定） |
| DB 接続 URL | jdbc:postgresql://localhost:5433/blog_demo |
| DB ユーザー | postgres |
| DDL 自動生成 | `spring.jpa.hibernate.ddl-auto=update` |
| 起動コマンド | `./mvnw spring-boot:run` |

> 補足: 本アプリは単一プロセス構成（組み込み Tomcat）で動作します。画像はアプリのリソースディレクトリ配下に保存されるため、運用時は外部ストレージへの分離を推奨します（[14_セキュリティ非機能設計.md](14_セキュリティ非機能設計.md) 参照）。
