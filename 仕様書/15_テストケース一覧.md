# テストケース一覧

| システム名 | Blog アプリケーション |
| --- | --- |
| テスト方式 | Spring Boot Test + MockMvc + Mockito（単体／結合） |

本書は、`src/test` 配下に実装された自動テストの観点を一覧化したものです。
（コメントアウトされているテストは「無効」として記載します）

## 1. テストクラス一覧

| No. | テストクラス | 対象 | Mock 対象 |
| --- | --- | --- | --- |
| 1 | BlogApplicationTests | アプリ起動（コンテキストロード） | - |
| 2 | BlogControllerTest | ブログ系機能（一覧/登録/編集/画像/削除/ログアウト） | BlogService |
| 3 | BlogControllerTest2 | 同上（セッション準備方法の派生版） | BlogService |
| 4 | UserLoginControllerTest | ログイン機能 | UserService |
| 5 | UserRegisterControllerTest | ユーザー登録機能 | UserService |

> 備考: `BlogControllerTest` と `BlogControllerTest2` はほぼ同一内容です。コントローラー分割後も、テストは URL ベース（MockMvc）で記述されているため、エンドポイントが不変であれば有効に動作します。

## 3. テスト実行方法

```bash
./mvnw test
```

## 4. 補足・改善提案

- **未実装／無効化テストの有効化**: 失敗系（記事なし・更新失敗・削除失敗）のテストがコメントアウトされています。分割後コントローラーの回帰確認のためにも有効化を推奨します。
- **Service / DAO 層の単体テスト**: 現状はコントローラー層中心です。`UserService` / `BlogService` のロジック（重複チェック、null 判定、画像差分判定など）に対する単体テストの追加を推奨します。
- **重複テストの整理**: `BlogControllerTest` と `BlogControllerTest2` は内容が重複しています。いずれかへの集約を検討してください。
