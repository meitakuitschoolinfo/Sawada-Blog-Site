# クラス図

本アプリのクラス構成と関連を示します（Mermaid 記法）。
機能分割後のコントローラー群、サービス、DAO、エンティティの依存関係を表します。

## 1. 全体クラス図

```mermaid
classDiagram
    direction LR

    %% ===== Controller 層 =====
    class UserRegisterController {
        -UserService userService
        +getUserRegisterPage() String
        +register(userName, email, password) String
    }
    class UserLoginController {
        -UserService userService
        -HttpSession session
        +getUserLoginPage() String
        +login(email, password) String
    }
    class BlogListController {
        -BlogService blogService
        -HttpSession session
        +getBlogListPage(model) String
    }
    class BlogRegisterController {
        -BlogService blogService
        -HttpSession session
        +getBlogRegisterPage(model) String
        +blogRegister(...) String
    }
    class BlogEditController {
        -BlogService blogService
        -HttpSession session
        +getBlogEditPage(blogId, model) String
        +blogUpdate(...) String
    }
    class BlogImageEditController {
        -BlogService blogService
        -HttpSession session
        +getBlogEditImagePage(blogId, model) String
        +blogImgUpdate(...) String
    }
    class BlogDeleteController {
        -BlogService blogService
        -HttpSession session
        +getBlogDeleteListPage(model) String
        +getBlogDeleteDetailPage(blogId, model) String
        +blogDelete(blogId, model) String
    }
    class LogoutController {
        -HttpSession session
        +logout() String
    }

    %% ===== Service 層 =====
    class UserService {
        -UserDao userDao
        +createAccount(userName, email, password) boolean
        +loginAccount(email, password) UserEntity
    }
    class BlogService {
        -BlogDao blogDao
        +findAllBlogPost(userId) List~BlogEntity~
        +createBlogPost(...) boolean
        +getBlogPost(blogId) BlogEntity
        +editBlogPost(...) boolean
        +editBlogImage(blogId, fileName, userId) boolean
        +deleteBlog(blogId) boolean
    }

    %% ===== DAO 層 =====
    class UserDao {
        <<interface>>
        +save(userEntity) UserEntity
        +findByEmail(email) UserEntity
        +findByEmailAndPassword(email, password) UserEntity
    }
    class BlogDao {
        <<interface>>
        +findByUserId(userId) List~BlogEntity~
        +save(blogEntity) BlogEntity
        +findByBlogTitleAndRegisterDate(title, date) BlogEntity
        +findByBlogId(blogId) BlogEntity
        +deleteByBlogId(blogId) List~BlogEntity~
    }
    class JpaRepository {
        <<interface>>
    }

    %% ===== Entity 層 =====
    class UserEntity {
        -Long userId
        -String userName
        -String email
        -String password
        -LocalDateTime registerDate
    }
    class BlogEntity {
        -Long blogId
        -String blogTitle
        -LocalDate registerDate
        -String blogImage
        -String blogDetail
        -String category
        -Long userId
    }

    %% ===== 依存関係 =====
    UserRegisterController --> UserService
    UserLoginController --> UserService
    BlogListController --> BlogService
    BlogRegisterController --> BlogService
    BlogEditController --> BlogService
    BlogImageEditController --> BlogService
    BlogDeleteController --> BlogService

    UserService --> UserDao
    BlogService --> BlogDao

    UserDao --|> JpaRepository
    BlogDao --|> JpaRepository

    UserDao ..> UserEntity
    BlogDao ..> BlogEntity
    UserService ..> UserEntity
    BlogService ..> BlogEntity

    UserEntity "1" --> "0..*" BlogEntity : userId で関連
```

## 2. レイヤーごとの責務

| クラス | レイヤー | 責務 |
| --- | --- | --- |
| `*Controller` | Controller | リクエスト受付・画面遷移・セッション参照 |
| `UserService` / `BlogService` | Service | 業務ロジック・判定・重複チェック |
| `UserDao` / `BlogDao` | DAO | DB アクセス（JpaRepository による自動実装） |
| `UserEntity` / `BlogEntity` | Entity | テーブルとのマッピング（Lombok でアクセサ自動生成） |

## 3. 補足

- 各 `Blog*Controller` は `BlogService` を共有して利用します（1 サービスを複数コントローラーが参照）。
- `UserDao` / `BlogDao` は `JpaRepository<Entity, Long>` を継承し、宣言したメソッドの実装は Spring Data JPA が生成します。
- `UserEntity` と `BlogEntity` は Lombok（`@Data` ほか）により getter / setter / コンストラクタが自動生成されます。
