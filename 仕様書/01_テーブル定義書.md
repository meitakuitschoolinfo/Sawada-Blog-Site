# テーブル定義書

| システム名 | Blog アプリケーション |
| --- | --- |
| DBMS | PostgreSQL |
| データベース名 | blog_demo |
| 生成方法 | JPA / Hibernate（`spring.jpa.hibernate.ddl-auto=update` による自動生成） |

本アプリで使用するテーブルは `users`（ユーザー）と `blogs`（ブログ記事）の 2 つです。
各テーブルの定義は、対応するエンティティクラスのアノテーションに基づきます。

---

## 1. users テーブル（ユーザー）

- 対応エンティティ: `blog.ex.model.entity.UserEntity`
- 概要: アプリの利用者アカウントを管理するテーブル。

| No. | 物理名 | 論理名 | 型 | PK | NOT NULL | 自動採番 | 説明 |
| --- | --- | --- | --- | :---: | :---: | :---: | --- |
| 1 | user_id | ユーザーID | BIGINT | ○ | ○ | ○ (AUTO) | 主キー。ユーザーを一意に識別する。 |
| 2 | user_name | ユーザー名 | VARCHAR | | ○ | | 画面に表示されるユーザー名。 |
| 3 | email | メールアドレス | VARCHAR | | ○ | | ログインIDとして使用。登録時に重複チェックを行う。 |
| 4 | password | パスワード | VARCHAR | | ○ | | ログイン用パスワード（学習用のため平文保存）。 |
| 5 | register_date | 登録日時 | TIMESTAMP | | ○ | | アカウント登録日時（`LocalDateTime`）。 |

### 補足
- `user_id` は `@GeneratedValue(strategy = GenerationType.AUTO)` により自動採番されます。
- `email` は一意であることを前提に運用されます（`createAccount` で重複チェック）。

---

## 2. blogs テーブル（ブログ記事）

- 対応エンティティ: `blog.ex.model.entity.BlogEntity`
- 概要: ユーザーが投稿したブログ記事を管理するテーブル。

| No. | 物理名 | 論理名 | 型 | PK | NOT NULL | 自動採番 | 説明 |
| --- | --- | --- | --- | :---: | :---: | :---: | --- |
| 1 | blog_id | ブログID | BIGINT | ○ | ○ | ○ (AUTO) | 主キー。ブログ記事を一意に識別する。 |
| 2 | blog_title | ブログタイトル | VARCHAR | | ○ | | 記事のタイトル。 |
| 3 | register_date | 登録日 | DATE | | ○ | | 記事の登録日（`LocalDate`、形式 `yyyy-MM-dd`）。 |
| 4 | blog_image | 画像名 | VARCHAR | | ○ | | 記事に紐づく画像ファイル名。 |
| 5 | blog_detail | ブログ詳細 | VARCHAR | | ○ | | 記事の本文。 |
| 6 | category | カテゴリー | VARCHAR | | ○ | | 記事のカテゴリー。 |
| 7 | user_id | ユーザーID | BIGINT | | | | 投稿者のユーザーID（`users.user_id` に対応する論理的な外部キー）。 |

### 補足
- `blog_id` は `@GeneratedValue(strategy = GenerationType.AUTO)` により自動採番されます。
- `user_id` は投稿者を表します。コード上で `users` テーブルと関連付けて利用しますが、JPA 上の外部キー制約（`@ManyToOne` 等）は定義されておらず、`user_id` カラムとして保持されています。
- 同一タイトル・同一登録日の記事は登録できません（`createBlogPost` で `findByBlogTitleAndRegisterDate` による重複チェック）。

---

## 3. テーブル間の関連

```
users (1) ──< (N) blogs
   user_id          user_id
```

- 1 人のユーザー（`users`）は複数のブログ記事（`blogs`）を持ちます。
- 関連は `blogs.user_id` = `users.user_id` の論理的な関連で表現されています。
